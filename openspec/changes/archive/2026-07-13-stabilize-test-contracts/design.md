## Context

`tests/__init__.py` 和 `tests/unit/__init__.py` 已存在，但 `tests/unit/archive/` 与 `tests/unit/recorded_video/` 不是 Python 包。pytest 默认导入模式把两个 `test_models.py` 都注册为顶层 `test_models`，因此第二个文件收集时命中已加载的第一个模块。两个文件分别执行时均通过，说明问题位于测试模块身份而非断言或运行时代码。

## Goals / Non-Goals

**Goals:**

- 为测试子目录建立唯一、稳定的模块身份。
- 保持单文件、子目录和全量测试执行结果一致。
- 用最小改动恢复全量测试门禁。

**Non-Goals:**

- 不修改生产模块以适配测试收集。
- 不重命名测试来回避冲突，不跳过或降级断言。
- 不在本 change 中处理 Ruff 格式债务。

## Decisions

1. 在存在重复测试文件名的子目录补充包边界，使模块名包含完整目录路径。相比全局切换 pytest `importlib` 模式，该方案影响更局部，也不会改变所有测试的导入语义。
2. 保留现有测试文件名和断言，将全量 `pytest --collect-only` 与 `pytest -q` 作为回归验证。单独运行两个冲突文件用于证明修复没有改变其内部行为。
3. 若扫描发现其他重复 basename，同样通过一致的测试包边界处理，不采用逐文件临时改名。

## Risks / Trade-offs

- [风险] 包初始化可能暴露测试中的相对导入假设。 -> 先运行收集，再运行受影响目录和全量测试定位差异。
- [风险] 当前功能分支仍在新增测试。 -> 验证以合并后的完整测试树为准，并保持修改仅限测试包结构。

## Migration Plan

1. 记录修复前的可重复收集失败。
2. 增加最小测试包边界并验证两个目标文件可共同收集。
3. 运行全量收集与测试；失败时只回滚包边界文件。
