## 背景

全量测试目前有九项确定性失败：测试把展示语言和开发者本地密钥当作稳定的运行时契约。这些失败会掩盖真实回归，并阻碍推送前验证得到可靠结果。

## 变更内容

- 以 Markdown 结构和报告数据定义报告流程测试预期，不依赖标题使用的自然语言。
- 为配置诊断和 DashScope 运行器包装脚本定义隔离的缺失 API Key 行为，包括既有退出码。
- 保留现有条件跳过；本变更不会将其改为失败，也不会移除环境保护条件。

## 能力

### 新增能力

- `language-independent-report-tests`：验证报告流程的语义区段和负载内容，不要求特定输出语言。
- `isolated-runtime-config-validation`：验证配置和 DashScope 包装脚本时，不允许开发者本地配置参与缺失密钥场景。

### 修改能力

- 无。

## 影响范围

- 受影响测试：报告验收流程、运行时配置诊断和 DashScope 实时运行器包装脚本。
- 若跨平台缺失密钥路径当前不可达，受影响实现将限于运行器脚本行为。
- 不改变公开 API、模型 Schema、报告渲染语言或条件跳过策略。
